RPF Archive

RPF (RAGE Package File) is an archive format used in Grand Theft Auto V to store game assets in a structured, compressed, and encrypted container. The format supports hierarchical organization of files and directories with multiple encryption schemes to protect game data.

File Format

Header

The RPF archive begins with a 16-byte header containing essential metadata:

OffsetSizeDescription
0x004 bytesMagic/Version - 0x52504637 ("RPF7" in ASCII) for GTA V
0x044 bytesTOC Size - Size of the Table of Contents in bytes
0x084 bytesEntry Count - Total number of entries (directories + files)
0x0C4 bytesEncryption Flag - Specifies the encryption scheme used

Example Header (Hex):

37 46 50 52  00 10 00 00  20 00 00 00  4F 50 45 4E

Breaking down this example:

  • 37 46 50 52 = "RPF7" magic signature
  • 00 10 00 00 = TOC size of 4096 bytes (0x1000)
  • 20 00 00 00 = 32 entries total (0x20)
  • 4F 50 45 4E = "OPEN" encryption flag (unencrypted)

Table of Contents (TOC)

The Table of Contents is located at offset 2048 (0x800) from the file start and contains the archive's directory structure.

  • Each entry is 16 bytes in length
  • Describes hierarchical organization of files and directories
  • Contains offsets, sizes, and pointers to the filename section

Directory Entry Structure

OffsetSizeDescription
0x003 bytesName Offset - Offset into filename section
0x031 byteEntry Type - 0x80 for directory
0x044 bytesFirst Entry Index - Index of first TOC entry in directory
0x084 bytesEntry Count - Number of entries in directory
0x0C4 bytesReserved - Duplicate count (alignment purposes)

Example Directory Entry (Hex):

00 00 00 80  01 00 00 00  05 00 00 00  05 00 00 00

Breaking down this example:

  • 00 00 00 = Name offset 0 (points to first filename)
  • 80 = Directory type flag
  • 01 00 00 00 = First entry index is 1
  • 05 00 00 00 = Contains 5 entries
  • 05 00 00 00 = Duplicate count (5)

File Entry Structure

OffsetSizeDescription
0x003 bytesName Offset - Offset into filename section
0x031 byteEntry Type - 0x00 for file
0x044 bytesData Offset - File data offset in 512-byte blocks
0x084 bytesCompressed Size - Size of compressed data in bytes
0x0C4 bytesUncompressed Size - Original file size in bytes

Example File Entry (Hex):

08 00 00 00  10 00 00 00  80 15 00 00  00 20 00 00

Breaking down this example:

  • 08 00 00 = Name offset 8 (points to filename at offset 8)
  • 00 = File type flag
  • 10 00 00 00 = Data starts at block 16 (16 × 512 = 8192 bytes offset)
  • 80 15 00 00 = Compressed size is 5504 bytes (0x1580)
  • 00 20 00 00 = Uncompressed size is 8192 bytes (0x2000)

Filename Section

The filename section immediately follows the TOC at offset: 2048 + (16 × number of entries)

  • Contains null-terminated UTF-8 strings
  • Stores all filenames and directory names
  • Name offsets in TOC entries reference this section

Example Filename Section (Hex):

72 6F 6F 74 00 74 65 78 74 75 72 65 73 00 6D 6F 
64 65 6C 73 00 73 63 72 69 70 74 73 00 6D 61 70 
2E 79 6D 61 70 00 63 61 72 2E 79 64 72 00

Decoded strings:

  • Offset 0: root\0
  • Offset 5: textures\0
  • Offset 14: models\0
  • Offset 21: scripts\0
  • Offset 29: map.ymap\0
  • Offset 38: car.ydr\0

Compression

Files within RPF archives use zlib compression with the deflate header removed:

  • If compressed size equals uncompressed size, the file is stored uncompressed
  • Compression reduces archive size while maintaining data integrity
  • Decompression is handled transparently during file access

Encryption

The RPF format implements multiple encryption schemes to protect game assets and prevent unauthorized modification.

Encryption Types

Flag ValueNameDescription
0x00000000NONENo encryption - raw or compressed data only
0x4E45504FOPENUnencrypted format (ASCII "OPEN") - modding-friendly
0x0FFFFFF9AESAES-256 encryption in ECB mode
0x0FEFFFFFNGRockstar's proprietary encryption method

Example Encryption Flags (Hex):

NONE:  00 00 00 00
OPEN:  4F 50 45 4E
AES:   F9 FF FF 0F
NG:    FF FF FE 0F

AES Encryption

  • Algorithm: Rijndael AES-256
  • Mode: ECB (Electronic Codebook)
  • Padding: None
  • Key Size: 32 bytes (256-bit)
  • Block Size: 16 bytes

Encrypted Components:

  • Table of Contents entries
  • Filename section
  • File content (in 16-byte blocks)

NG Encryption (Proprietary)

Rockstar's proprietary "NG" encryption is significantly more complex than AES:

  • Multi-round encryption with 17 rounds
  • Each round uses 4 subkeys (68 total)
  • Large precomputed lookup tables
  • Dynamic key derivation per file
  • File-specific encryption ties security to file identity

File-Level Encryption

  • Encryption flag in file entry indicates encrypted status
  • NG-encrypted files use file-specific derived keys
  • Key derivation combines filename hash with file size

Technical Implementation

Data Access

  1. Parse RPF header to determine encryption type
  2. Decrypt TOC if encryption is enabled
  3. Navigate directory structure
  4. Locate target file entry and retrieve data offset
  5. Decrypt and decompress file data as needed

Block Alignment

  • File data offsets are in 512-byte blocks
  • AES works on 16-byte boundaries
  • NG can handle partial blocks
